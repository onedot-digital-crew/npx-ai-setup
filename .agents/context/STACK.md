# Stack

## Runtime & Distribution
- **Language:** Bash (POSIX sh compatible)
- **Package Manager:** npm (published as `@onedot/ai-setup`)
- **Version Control:** Git with hooks and worktrees
- **Node Version:** Not required (bash CLI only)

## Framework & Key Dependencies
- **Core Runtime:** bash 4.0+, git 2.13+ (worktree support)
- **Stdlib:** bash built-ins only; no external CLI dependencies beyond git
- **Version Control Hooks:** git hooks (pre-commit, post-commit, pre-push)
- **Module System:** Custom source_lib pattern (`lib/` folder with . sourcing)

## Build & Tooling
- **Build System:** None (pure bash, no compilation)
- **Linting:** shellcheck (recommended, not enforced)
- **Testing:** Bash smoke tests (`tests/smoke.sh`)
- **Distribution:** npm tarball (files array in package.json)

## Key Patterns & Libraries to Avoid
- **Do use:** Modular bash with lib/ modules, explicit error checking (`|| return 1`), quoted variable expansions
- **Avoid:** bashisms that break in sh (arrays, regex =~, declare -A), unquoted variables, silent failures, sed/awk for text processing (use built-in bash)
- **Avoid creating:** External tool dependencies (no curl/jq/yq requirements)

## Template & Context Generation
- **Templates:** Copied verbatim from `templates/` (not generated)
- **Context Files:** STACK.md, ARCHITECTURE.md, CONVENTIONS.md generated by Claude
- **Idempotency:** All operations must be safe to run multiple times
