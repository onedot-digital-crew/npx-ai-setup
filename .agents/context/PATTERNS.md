# Patterns

> Generated by /analyze — do not edit manually. Re-run /analyze to refresh.

## Entry Points

- `bin/ai-setup.sh` — Projekt-AI-Setup (Hooks, Rules, Skills, Context)
- `bin/global-setup.sh` — Globaler Workstation-Setup (Dependencies, CLI-Tools, Settings)

## Data Flow

1. `bin/ai-setup.sh` → `lib/_loader.sh` → alle `lib/*.sh` Module
2. `lib/core.sh` baut `TEMPLATE_MAP` → steuert Install/Update
3. `lib/json.sh` abstrahiert jq/Node.js für JSON-Operationen
4. `lib/setup.sh` installiert mit Checksum-basiertem Smart-Update
5. `lib/generate.sh` orchestriert parallele Claude-CLI-Aufrufe
6. `lib/update.sh` verwaltet Versionen, Migrations, Smart-Updates

## Module Boundaries

| Layer | Module | Verantwortung |
|-------|--------|---------------|
| Entry | `bin/ai-setup.sh`, `bin/global-setup.sh` | CLI-Einstieg, Flow-Steuerung |
| Core | `lib/core.sh`, `lib/json.sh`, `lib/_loader.sh` | Template-Map, JSON-Ops, Loading |
| Install | `lib/setup.sh`, `lib/setup-skills.sh`, `lib/setup-compat.sh` | Datei-Installation, Skills, Compat |
| Update | `lib/update.sh`, `lib/migrate.sh`, `lib/migrations/*.sh` | Version-Check, Smart-Update |
| Support | `lib/tui.sh`, `lib/process.sh`, `lib/detect.sh` | Terminal-UI, Prozess-Mgmt |

| Hooks | `.claude/hooks/*.sh` (16) | Claude Code Lifecycle-Hooks |
| Scripts | `.claude/scripts/*.sh` (15) | Prep-Scripts für Slash-Commands |
| Templates | `templates/` | 1:1 Mirror von `.claude/` |

## Key Abstractions

- `_install_or_update_file()` — zentrale Install-Funktion mit Checksum-Vergleich
- `prep-lib.sh` → `has()`, `rtk_or_raw()`, `git_guard()`, `main_branch()`
- `TEMPLATE_MAP` Array — dynamisch aufgebaut, steuert Install/Update
- Exit-Code-Konvention: 0=grün, 1=Fehler, 2=Issues (alle Prep-Scripts)
- Dual JSON-Engine: jq bevorzugt, Node.js als Fallback (json.sh)

## Established Patterns

- **Prep-Script Pattern**: Shell sammelt Daten vor Claude-Kontext → Zero Tokens auf Green Path
- **Checksum-Guard**: `.ai-setup.json` speichert Checksums → erkennt User-Modifikationen
- **Hook Convention**: stdout=JSON für Claude, stderr=Warnings für User, exit 0=allow, exit 2=block
- **Template Sync**: `templates/` ist Quelle der Wahrheit, `.claude/` wird daraus installiert
- **Parallel Execution**: `wait_parallel()` in process.sh für gleichzeitige Claude-CLI-Aufrufe
