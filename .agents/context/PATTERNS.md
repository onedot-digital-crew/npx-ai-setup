# Patterns

> Generated by /analyze — do not edit manually. Re-run /analyze to refresh.

## Architecture

Entry points in `bin/` source `lib/_loader.sh`, which loads all `lib/*.sh` modules. Data flows: detect stack → build `TEMPLATE_MAP` → copy templates → generate context (parallel Claude calls) → install skills/hooks → verify.

```
bin/             Entry: ai-setup.sh (project), global-setup.sh (workstation)
lib/             Modules loaded via source_lib; bash 3.2 compatible throughout
templates/       Source of truth for all installed config files
.claude/         Active config for this repo (hooks, skills, scripts, rules)
.agents/context/ Generated context (STACK, ARCHITECTURE, CONVENTIONS, AUDIT, SUMMARY)
specs/           Feature specs; completed/ holds finished work
tests/           smoke.sh + integration.sh + claude-runtime.sh
```

## Module Layers

| Layer | Modules | Responsibility |
|-------|---------|---------------|
| Entry | `bin/ai-setup.sh`, `bin/global-setup.sh` | CLI flow, arg parsing |
| Core | `lib/core.sh`, `lib/json.sh`, `lib/_loader.sh` | TEMPLATE_MAP, JSON ops, module sourcing |
| Install | `lib/setup.sh`, `lib/setup-skills.sh`, `lib/setup-compat.sh` | File install with checksum + smart-merge |
| Update | `lib/update.sh`, `lib/migrate.sh`, `lib/migrations/*.sh` | Version detection, migrations |
| Support | `lib/tui.sh`, `lib/process.sh`, `lib/detect.sh` | Terminal UI, parallel exec, stack detection |
| Plugins | `lib/plugins.sh`, `lib/skills.sh`, `lib/boilerplate.sh` | Plugin/skill/boilerplate install |

## Patterns

**Checksum-Guard:** `.ai-setup.json` stores installed file checksums. `_install_or_update_file()` (`lib/setup.sh:68`) compares template vs. installed vs. stored checksum — skips identical, auto-updates unmodified, smart-merges user-edited files.

**Smart-Merge:** `_smart_merge_file()` (`lib/setup.sh:9`) invokes `claude -p` with Haiku on user-modified `.md` files. Preserves local additions while applying template updates. Falls back to skip if Claude CLI unavailable.

**Dual JSON-Engine:** `lib/json.sh` detects `jq` at load time; all `_json_*` functions use jq or Node.js fallback. Callers never branch on engine. Six hooks and `lib/update.sh` bypass this abstraction and call `jq` directly — inconsistency tracked as R3/R4.

**Hook Convention:** Hooks read JSON from stdin via `jq -r`. `exit 0` = allow, `exit 2` = block with stderr guidance. `tool-redirect.sh` is the canonical example: parse `tool_name` → match → emit stderr → exit 2.

**Template-Sync:** `templates/` is the source of truth. `build_template_map()` (`lib/core.sh:16`) dynamically maps `templates/X/` paths to `.claude/X/`. Skills and commands are excluded from the generic map and handled separately by `install_skills()`.

**Parallel Execution:** `wait_parallel()` in `lib/process.sh` runs Claude CLI calls as background jobs. Used in `run_generation()` (`lib/generate.sh`) for concurrent STACK.md + ARCHITECTURE.md + CONVENTIONS.md generation.

**build-summary.sh:** (`.claude/scripts/build-summary.sh`) Aggregates YAML frontmatter `abstract:` and `sections:` from AUDIT.md + ARCHITECTURE.md into SUMMARY.md (max 50 lines). Run manually via `bash .claude/scripts/build-summary.sh`. Frontmatter fields in these files drive CLAUDE.md context — keeping them accurate is required.

## Conventions

- **bash 3.2 compat (macOS):** No `declare -A`, no `=~`, no process substitution in conditions. Use `case` for string matching; `eval` for by-reference variable updates.
- **Module sourcing:** `source_lib "module.sh"` not relative paths. Modules declare deps in header comments.
- **Error reporting:** `tui_error`/`tui_warn` to stderr; `tui_success` to stdout. Never raw `echo` for errors.
- **Exit codes:** 0=success, 1=error, 2=block (hooks only). Functions use `return 1` not `exit 1`.
- **Idempotency:** All install operations safe to re-run. Checksum-guard prevents accidental overwrites.
- **Quoting:** Always `"$var"`, `"${var}"`, `"$@"`. Unquoted expansions fail shellcheck.
